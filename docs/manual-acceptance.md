# MVP 人工浏览器验收矩阵

## 完成规则与环境

先在镜像预热后运行 `./blog check`，确认成功且不超过 60 秒。然后使用当前稳定版 Safari 与 Chrome，分别在 1440×900 桌面视口和 390×844 移动模拟视口执行下表；每个视口都覆盖浅色与深色模式。自动检查与本矩阵必须全部通过，任何一项失败都表示 MVP 未完成。

生产预览的响应门槛：本地导航应在 1 秒内呈现主要内容；搜索在约 200 ms 防抖后 500 ms 内显示结果或明确加载状态，并在 1 秒内完成首批 20 条结果。

## 固定操作流程

1. 打开主页，验证 21 条以上正式 fixture 的日期倒序、同日完整文件名次序、分页和标签计数。
2. 切换“显示草稿”，验证草稿、草稿专属标签和相邻详情导航；关闭后验证它们从集合与默认搜索消失。
3. 在搜索页依次搜索固定的标题、摘要、标签和正文命中词；确认代码与附件内容不作为命中来源，正式与草稿结果不重复。
4. 验证空查询、单字符、加载、无结果和不可用状态，并确认切换草稿保留查询但重置批次。
5. 打开长文详情，验证二三级目录、代码语言、横向滚动、复制成功/失败反馈、附件链接和 HTTPS 外链。
6. 仅使用键盘重复核心路径；逐项检查焦点可见性、视觉顺序、菜单关闭后的焦点恢复。
7. 在浅色、深色和 200% 缩放下复查；用对比度工具验证 WCAG 2.2 AA。
8. 分别调整至 390×844、768 px、1024 px 与 1439 px，确认没有整页水平溢出且主要内容可用。
9. 记录浏览器版本、日期、每个方框结果与失败证据。所有方框勾选后，人工门槛才通过。
